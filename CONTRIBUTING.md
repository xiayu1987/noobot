# Contributing

感谢贡献！

## 提交流程

1. Fork 并新建分支（`feat/...`、`fix/...`）
2. 本地运行并确保能正常启动：
   ```bash
   ./start.sh
   ```
3. 提交 PR，描述清楚变更动机、影响范围、验证方式

## 代码建议

- 保持改动小而清晰
- 不提交构建产物和运行时文件（见 `.gitignore`）
- 涉及配置项时，同步更新文档与 `.env.example`

## Commit 建议

推荐使用：

- `feat: ...`
- `fix: ...`
- `docs: ...`
- `refactor: ...`
